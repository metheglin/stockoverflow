# テスティング規約

## 基本方針

- テストフレームワークは **RSpec** を使用する

## テストの種類ごとの方針

### app/models

- モデル内に追加されたインスタンスメソッド及びクラスメソッドの動作をテストすること
- バリデーション、スコープ、コールバックのテストは一切書かないこと
- なるべくDB読み書きをおこなわず、必要な値が代入された手元のインスタンスのみによるテストが完結するようメソッドを設計すること

### app/lib

- 原則テストしやすい設計を維持し、テストをおこなうこと
- APIクライアントなどにおいて、APIKEYを指定されているものは原則API呼び出しを実際におこなうテストを記述すること
  - ただし、基本的には影響が限定的なデータの一覧・詳細や、ログイン機能のみをおこなうこと
  - 影響が不透明な書き込みを伴うAPI呼び出しはテストに記述しないこと

### app/jobs

- モデル内に切り出されているメソッドがある場合、そのメソッドに対してテストを記述すること
- 実際にジョブを稼働させるテストは記述しないこと

### app/controllers, app/views

- 原則テストを一切記述しないこと
