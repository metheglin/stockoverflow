# PLAN: 実装優先度ロードマップの策定

## 概要

93件を超えるpending TODOが存在するが、実装順序・依存関係・優先度が整理されていない。CLAUDE.mdに記載された3つのユースケースを実現可能にすることを最優先目標として、全TODOの優先度を整理し、フェーズ分けと依存関係を明確にした実装ロードマップを策定する。

## 背景・動機

- 93件のpending TODOが未整理のまま蓄積されており、次に何を実装すべきかの判断が困難
- 各TODOの依存関係が暗黙的で、順序を誤ると手戻りが発生するリスクがある
- プロジェクトの目標（3つのユースケースの実現）から逆算した優先順位付けがなされていない
- 高度な指標（Piotroski, Altman Z-Score等）のTODOが多いが、基盤となる分析クエリレイヤーやRakeタスクが未実装

## 計画策定の観点

### 1. 3つのユースケースから逆算したクリティカルパス

以下の3つのユースケースを「動かせる状態」にするために最低限必要なTODOを特定し、それを最優先フェーズとする:

1. 6期連続増収増益企業の一覧（増収率順ソート）
2. 営業CF+/投資CF-の企業のうちフリーCF転換企業の検出
3. 企業の業績飛躍前の財務変化の調査

### 2. フェーズ分け

- **Phase 0（基盤）**: 他の多くのTODOの前提となる基盤タスク
  - 分析クエリレイヤー、期間ナビゲーション、Company検索など
- **Phase 1（運用可能化）**: システムを日常的に運用可能にするタスク
  - Rakeタスク、パイプラインオーケストレーション、ステータス確認
- **Phase 2（分析強化）**: 分析機能の充実
  - 各種財務指標、スクリーニング、ランキング
- **Phase 3（高度分析）**: 高度な分析・可視化
  - DCF、Magic Formula、Web UI等
- **Phase X（いつでも可能）**: 他のフェーズに依存しないタスク
  - テスト強化、パフォーマンス改善等

### 3. 依存関係の明確化

各TODOについて:
- 前提となるTODO（これが完了しないと着手できない）
- 後続のTODO（このTODOの完了により着手可能になる）
- 並行実施可能なTODO（依存関係がなく同時に着手可能）

### 4. 不要・重複TODOの特定

- 類似目的のTODOが複数存在する場合の統合提案
- スコープが曖昧で実装判断ができないTODOの改善提案
- プロジェクト目標に対して優先度が極めて低いTODOの識別

- 各既存TODOファイルへのフェーズ・優先度のタグ付け提案
