# dev_financial_anomaly_detection

## 概要

企業の財務指標における異常値（一時的要因・特殊項目・会計変更等による歪み）を自動検出し、フラグ付けする。

## 背景・目的

スクリーニングや企業分析において、数値の大幅な変動が「本質的な業績改善/悪化」なのか「一時的要因（特別利益・減損・会計方針変更等）」なのかを区別することが重要である。

例:
- 純利益が前年比+500%だが、固定資産売却益による一時的なもの
- 営業利益率が急落したが、のれん減損による一時的なもの
- 売上が急増したが、決算期変更による13ヶ月決算

これらを自動検出することで、スクリーニング結果のノイズを除去し、分析の精度を向上させる。

## 実装内容

- financial_metrics の data_json 内に anomaly フラグと anomaly_details を格納
- 検出ロジック:
  1. **統計的異常値検出**: 直近5期の指標の平均・標準偏差から2σ以上の乖離を検出
     - 対象: 売上YoY, 営業利益YoY, 純利益YoY, 各利益率
  2. **利益指標間の乖離検出**: 営業利益は安定なのに純利益だけ急変（特別損益の兆候）
     - operating_income_yoy と net_income_yoy の差が大きい場合にフラグ
  3. **CF乖離検出**: 利益は増加なのに営業CFは減少（利益の質の問題）
     - net_income増加 かつ operating_cf減少 をフラグ
  4. **決算期異常検出**: fiscal_year_start〜fiscal_year_end の期間が通常（12ヶ月）でない場合
- anomaly_details に検出理由をJSON形式で記録

## テスト

- 純利益だけが異常に変動し、営業利益は安定している場合に、特別損益フラグが立つことを検証
- 5期分のデータから2σ以上の乖離が正しく検出されることを検証
- データが不十分な場合（2期未満）にエラーにならないことを検証
