# 株価テクニカル指標の算出基盤

## 概要

DailyQuoteテーブルに蓄積された株価データから基本的なテクニカル指標を算出し、ファンダメンタル分析と組み合わせたスクリーニングを可能にする。

## 背景

現在DailyQuoteにOHLCVデータが蓄積されているが、移動平均・出来高トレンドなどのテクニカル指標は一切算出されていない。「業績が良く、株価もトレンド転換した企業」のようなファンダメンタル+テクニカルの複合スクリーニングは投資判断において有用。

## 実装内容

### 1. DailyQuoteモデルへの算出メソッド追加

以下のクラスメソッドを追加する（DBには保存せず、都度計算または一時的にメモリに保持）:

- `get_moving_averages(quotes, windows: [5, 25, 75, 200])` → 各ウィンドウ幅の移動平均を算出
- `get_volume_average(quotes, window: 25)` → 出来高移動平均
- `get_price_position(price, ma_short, ma_long)` → 移動平均との乖離率（現在株価が短期MA/長期MAの何%上にあるか）

### 2. CompanyモデルまたはQueryObjectへのスクリーニング支援

- ある基準日における各企業の株価ポジション（移動平均線との位置関係）を取得
- 「25日MAが75日MAをゴールデンクロスした企業」の検出
- 「出来高が25日平均の2倍以上の日が直近にある企業」の検出

### 3. daily_quotes の data_json 拡張（オプション）

頻繁に利用するテクニカル指標は、CalculateFinancialMetricsJob的なジョブで事前計算しdata_jsonに保存することを検討:

```json
{
  "ma5": 1520.0,
  "ma25": 1480.0,
  "ma75": 1430.0,
  "volume_ma25": 150000
}
```

ただし、DailyQuoteのレコード数が非常に多いため（全上場企業×営業日数）、パフォーマンスを考慮し、必要に応じてこの方式を採用する。

### 4. テスト

- get_moving_averages: 正常系（十分なデータ）、不十分なデータ、空配列
- get_price_position: 乖離率の算出テスト
- ゴールデンクロス/デッドクロス検出のテスト
