# PLAN: Web API設計

## 背景

データの蓄積・分析指標の算出基盤は完成しているが、データにアクセスする手段が一切存在しない。`config/routes.rb` にはヘルスチェックのみ定義されている状態。

個人用データベースとして利用するにあたり、分析結果やデータにプログラマティックにアクセスできるREST APIを整備する必要がある。

## 設計対象

### 企業情報API

- 企業一覧（上場企業のフィルタリング、業種・市場区分での絞り込み）
- 企業詳細（企業基本情報 + 最新の財務指標サマリー）

### 財務データAPI

- 特定企業の財務数値一覧（`financial_values` の時系列データ）
- 特定企業の分析指標一覧（`financial_metrics` の時系列データ）
- 特定企業の株価データ（`daily_quotes` の期間指定取得）

### 分析・スクリーニングAPI

- 分析クエリレイヤー設計で定義されるクエリをAPI経由で実行
- 連続増収増益企業スクリーニング
- キャッシュフロー転換企業スクリーニング
- 汎用フィルタ・ソートによるスクリーニング

### ページネーション・レスポンス形式

- カーソルベース or オフセットベースのページネーション
- JSON:APIやカスタム形式などレスポンス形式の統一
- Serializer設計（`app/serializers/`）

## 設計すべき項目

1. **ルーティング設計**: RESTfulなURL設計、名前空間（`/api/v1/`）
2. **コントローラー設計**: 各エンドポイントの入出力仕様
3. **シリアライザー設計**: レスポンスJSON構造の定義
4. **認証**: 個人用途のため簡易的な認証で十分（API Token等）
5. **エラーハンドリング**: 統一的なエラーレスポンス形式

## 前提

- 分析クエリレイヤー設計（`plan_analysis_query_layer`）が先に完了していること
- APIの利用者は開発者本人を想定（管理画面UIは別途検討）
