# 業績飛躍前パターン分析の設計

## 概要

「ある企業の業績が飛躍し始める直前にどのような決算・財務上の変化があったか」を分析する機能を設計する。
プロジェクトの中核ユースケースの1つに直接対応する。

## 背景

- プロジェクト仕様に明記されたユースケース:
  > （数年にわたって素晴らしい成果を出した）ある企業の業績が飛躍し始める直前にどのような決算・財務上の変化があったかを調べる
- 既存TODOの `plan_trend_turning_point_detection` は「転換点の検出」に焦点を当てているが、本タスクは「飛躍期間の特定 → その直前期の分析」という逆方向のアプローチ
- 単なるスクリーニングではなく、過去の事例研究型の分析機能

## 設計対象

### 1. 飛躍（ブレイクスルー）の定義と検出

- 飛躍とみなす条件の定義
  - 例: 3年以上にわたって売上YoY 20%以上かつ営業利益YoY 30%以上
  - 例: 株価が3年で3倍以上
  - 例: ROEが5%以下から15%以上に改善
- 飛躍開始年度の特定アルゴリズム
  - 連続成長の開始点を特定

### 2. 飛躍直前期の財務特徴量抽出

- 飛躍開始の1〜3期前の財務データを自動抽出し、以下の特徴をまとめる
  - 収益性の変化（利益率の改善が先行していたか）
  - CF構造の変化（投資CFの増加が先行していたか）
  - B/S構造の変化（レバレッジ変化、自己資本比率変化）
  - 成長率の加速パターン（売上成長率の変化の変化＝二次微分）
  - バリュエーションの状態（飛躍前のPER/PBRは割安だったか）

### 3. パターン類型化

- 複数の飛躍企業の飛躍前パターンを比較し、共通パターンを類型化
  - "投資先行型": 投資CF増大 → 売上成長加速
  - "収益性改善型": 利益率改善 → 成長加速
  - "レバレッジ活用型": 負債増加 → 事業拡大 → 成長加速
  - "隠れ成長型": 低バリュエーション + じわじわ改善 → 市場が気づき急伸

### 4. 現在の企業への適用

- 過去の飛躍パターンに類似する現在の企業を検出する「パターンマッチング」機能
- 「この企業は過去のX社の飛躍前と似た財務パターンを示している」というインサイト

## 検討事項

- ブレイクスルー定義のパラメータ化（ユーザーが条件を変更できる設計）
- 必要なデータ年数（最低5年分の過去データが必要か）
- パターンマッチングのアルゴリズム（コサイン類似度、ユークリッド距離、etc.）
- analysis_query_layerとの連携
- 出力フォーマット（レポート型 or テーブル型）
