# WEBダッシュボード

分析のWEBダッシュボード開発を検討する。

今のところ大きく分けて2種類のダッシュボード機能が求められている。  
あくまでもダッシュボードの要件として必要な開発をおこなうこと。検索条件を満たすために新たな情報の取得ロジックや集計ロジックの開発は本TODOではおこなわないこと。

(1) 検索ダッシュボード

様々な指標で企業を検索・フィルタすることを目的としたダッシュボード

重要な要件を列挙する。

- 軸をいくつも増やしてAND/ORでつなげたり、動的な検索条件を指定できる
- 設定した検索条件を名前をつけて保存して次回再利用できる
- 可能であれば、名前をつけて保存した条件を他の条件と組み合わせてつかうこともできるようにしたい
- 並び替えは今のところ必須ではない。どちらかというと並び替えではなく、「2025年のPER上位10件の会社」「2025年のPER上位50件の会社」のような「条件」として検索できるようにしたい
- 検索条件に関連した指標を結果一覧に含める。複数ある場合重要な数個を一覧に表示。一覧では、あとはティッカーコードと会社名だけでよい
- デフォルトダークモードとする

(2) 詳細ダッシュボード

ある特定の企業について、あらゆる指標を可視化することを目的としたダッシュボード

重要な要件を列挙する。

- 可能なかぎりグラフをつかって表示。ただ、表などで数値情報も併記すること
- いろんな軸で比較ができるようなUIを検討したい
- デフォルトダークモードとする
