# JQUANTS APIクライアント設計

## 概要

J-Quants API を利用して株価情報・財務情報・企業情報を取得するためのAPIクライアントを設計する。
`app/lib/` に配置し、コーディング規約の「汎用性と利便性」の方針に従って設計する。

## 調査・設計の対象

### JQUANTS APIの仕様調査

- J-Quants API の利用可能なエンドポイント
- 認証フロー（リフレッシュトークン → IDトークン）
  - APIキー: `Rails.application.credentials.jquants.api_key`
- 利用可能なデータ種別
  - 上場銘柄一覧
  - 株価四本値
  - 財務情報
  - 決算発表日
- レート制限・プラン別制約

### クライアント設計

- `app/lib/jquants_api.rb` のクラス設計
  - コーディング規約に従い、`api_key` を引数で指定可能にしつつ、便利メソッドも提供
  - Faraday を使ったHTTPクライアント実装方針
  - トークンリフレッシュの仕組み
  - エラーハンドリング方針

### EDINETとのデータ連携

- 企業の紐づけ方法（EDINETコード ↔ 証券コード）
- データ重複の扱い（両APIから取得可能な財務データの優先度）
