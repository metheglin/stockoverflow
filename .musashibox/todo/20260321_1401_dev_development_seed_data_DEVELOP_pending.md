# DEVELOP: 開発用シードデータの整備

## 概要

`db/seeds.rb` にリアルな財務データのサンプルを生成するコードを実装し、API接続なしで画面開発・分析機能開発・デバッグを進められる環境を整備する。

## 背景・動機

現在、開発用のデータを用意するにはEDINET/JQUANTS APIへの実際の接続が必要。以下の問題がある:

- APIキーがない環境（CI、新規開発者）ではデータが空の状態でしか開発できない
- Web UI/ダッシュボード開発（pending TODO: plan_web_api, plan_web_dashboard）にはある程度のデータ量が必須
- スクリーニング機能の開発・テストには多様なパターンの企業データが必要
- 分析クエリの動作確認にリアルなデータ分布が必要

### `improve_test_data_factories`（FactoryBot）との違い

FactoryBotはRSpecテスト内で一時的なインスタンスを生成するためのもの。本TODOは、開発用データベースに永続化されるサンプルデータセットを生成するもので、目的が異なる。

## 実装方針

### db/seeds.rb

```ruby
# rake db:seed で実行
# 冪等性を確保し、何度実行しても同じ結果になるようにする
```

### 生成するデータ

#### 企業マスタ（20社程度）

多様なパターンをカバーする架空の企業群:

- **連続増収増益企業** (3社): 6期以上の連続成長。ユースケース1の検証用
- **CF転換企業** (2社): 営業CF+/投資CF-でFCFがマイナスからプラスに転じた企業。ユースケース2の検証用
- **飛躍企業** (2社): 特定の期から急成長が始まった企業。ユースケース3の検証用
- **安定成熟企業** (3社): 横ばいの売上、高い利益率
- **景気循環企業** (2社): 好不調の波がある
- **赤字転落企業** (2社): 黒字→赤字の推移
- **新興高成長企業** (2社): 赤字だが急成長
- **高配当企業** (2社): 高い配当利回り
- **一般的な企業** (2社): 平均的な数値

#### 各企業に対する生成データ

- **FinancialReport**: 直近5〜7期分（年次）
- **FinancialValue**: 各レポートに対応する財務数値（連結ベース）
- **FinancialMetric**: 各FinancialValueに対応する計算済み指標
- **DailyQuote**: 直近30営業日分の日次株価
- **ApplicationProperty**: 同期状態の初期値

#### セクター・市場の分布

- 情報・通信業、製造業、サービス業、小売業などを分散配置
- プライム市場、スタンダード市場、グロース市場を混在

### データの現実性

- 売上高・利益の規模感は中堅〜大企業をモデルにする
- YoY成長率、利益率、ROE等は日本企業の実際の分布に近い値を設定
- 株価水準はPER/PBRから逆算して設定

### 冪等性

- `find_or_create_by` を活用し、何度実行しても重複しない
- 証券コード `99001`〜`99020` のような明らかにサンプルとわかるコードを使用

## テスト

- シードデータのテストは不要（`db:seed` の実行確認のみ）
- ただし、生成されたデータで `CalculateFinancialMetricsJob` が正常動作することは手動で確認
