# PLAN: 動的ピアグループ発見・類似企業クラスタリングの設計

## 概要

固定的な17業種/33業種分類に依存しない、財務プロファイルに基づく動的なピアグループ発見・類似企業検索の仕組みを設計する。「企業Xに似た企業を見つける」機能の基盤。

## 背景・動機

現在のセクター分析基盤（`dev_sector_analysis_foundation`）は33業種/17業種の固定分類に基づいてセクター統計を算出し、個別企業のセクター内ポジションを評価する。しかし、固定分類には以下の限界がある:

### 固定分類の限界

1. **異質な企業の同居**: 「情報・通信業」には通信キャリア（NTT等）もSaaSスタートアップ（ラクスル等）も含まれるが、財務構造は全く異なる
2. **類似企業の分離**: 医療IT企業と一般IT企業は異なるセクターに分類されるが、ビジネスモデルは類似する場合がある
3. **規模の無視**: 同セクター内に時価総額10兆円の企業と100億円の企業が混在し、統計量が歪む
4. **国際的なセクター分類との不整合**: GICS等のグローバル分類との比較が困難

### 動的ピアグループの価値

- 「この企業と似た成長パターン・収益構造の企業はどこか」を発見
- より意味のあるバリュエーション比較（似た企業群の平均PERとの比較）
- プロジェクト目標「飛躍前兆」: 過去に飛躍した企業と現在似ているプロファイルの企業を発見

## 設計すべき項目

### 1. 類似度指標の設計

企業間の「類似度」を定量化するためのメトリクス選定と距離関数の設計:

#### 候補となる特徴量

| カテゴリ | 特徴量 | 意味 |
|---------|--------|------|
| 規模 | 売上高（対数） | 事業規模の類似性 |
| 規模 | 時価総額（対数） | 市場評価の類似性 |
| 成長性 | revenue_yoy | 成長段階の類似性 |
| 成長性 | net_income_yoy | 利益成長の類似性 |
| 収益性 | operating_margin | ビジネスモデルの類似性 |
| 収益性 | roe | 資本効率の類似性 |
| 財務健全性 | equity_ratio | 財務レバレッジの類似性 |
| CF | free_cf / net_sales | キャッシュ創出力の類似性 |

#### 距離関数

- ユークリッド距離（正規化後）
- コサイン類似度（特徴量ベクトルの方向の類似性）
- マハラノビス距離（相関を考慮）

### 2. クラスタリング手法の検討

- **K-means**: シンプルだがRubyでの実装コストが低い。クラスタ数の事前指定が必要
- **階層的クラスタリング**: デンドログラムによる可視化が可能だがO(n^2)の計算コスト
- **k近傍法（kNN）**: 「企業Xに似た上位N社」の検索に最適。クラスタリングではなく類似検索
- 推奨: kNN方式を軸にし、特徴量ベクトルの距離で類似企業を検索

### 3. データモデルの検討

#### 案A: オンデマンド計算

```ruby
class Company::PeerDiscoveryQuery
  def initialize(company:, feature_keys: DEFAULT_FEATURES, limit: 10)
  def execute
    # 全企業の特徴量ベクトルを構築し、対象企業との距離を計算してソート
  end
end
```

- メリット: テーブル追加不要、常に最新のデータで計算
- デメリット: 毎回全企業を走査（約4000社 x 8特徴量 = 軽量）

#### 案B: 事前計算テーブル

```
company_peers テーブル
- company_id (FK)
- peer_company_id (FK)
- similarity_score (decimal)
- feature_set (string) -- 使用した特徴量セット名
- calculated_on (date)
```

- メリット: 検索が高速、ランキングの安定性
- デメリット: 4000^2 = 1600万件の組み合わせ（上位N件に制限すれば4000 x N件）

### 4. 特徴量の正規化方法

- 各特徴量をz-score正規化（平均0、標準偏差1）
- または全企業のパーセンタイルランクに変換（0-1）
- 金融セクターの除外/包含オプション

### 5. ユースケースインターフェース

```ruby
# 基本的な類似企業検索
query = Company::PeerDiscoveryQuery.new(
  company: target_company,
  limit: 10,
)
results = query.execute
# => [{ company: ..., similarity: 0.92, features: {...} }, ...]

# 特徴量カスタマイズ
query = Company::PeerDiscoveryQuery.new(
  company: target_company,
  feature_keys: [:operating_margin, :revenue_yoy, :roe],
  limit: 20,
)

# セクター横断の類似企業発見
# → 「この医療IT企業と財務プロファイルが似た他セクターの企業は？」
```
