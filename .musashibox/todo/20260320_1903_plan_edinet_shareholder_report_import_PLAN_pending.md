# 大量保有報告書インポートの計画

## 概要

EDINETで公開される大量保有報告書（doc_type_code: 140系とは異なる体系）のインポートを計画する。
機関投資家の保有動向は、企業分析における重要なシグナルであり、スクリーニング条件として活用できる。

## 背景・動機

- 現在のEDINETインポートは有価証券報告書・四半期報告書（doc_type_code 120,130,160,170）のみ対象
- 大量保有報告書は発行済株式の5%以上を保有する株主の変動を開示するもので、以下の分析に有用:
  - 著名な機関投資家（アクティビスト等）の参入・撤退の検出
  - 保有比率の増減トレンドの追跡
  - 「飛躍し始める直前」の企業における外部株主の動向把握
- プロジェクトの目的である「注目すべき企業や情報を一覧できるシステム」に直結する

## 計画すべき事項

### 1. EDINETの大量保有報告書のデータ構造調査
- docTypeCodeの特定（大量保有報告書: 010, 変更報告書: 020 等を確認）
- 取得可能なデータフィールドの調査
- XBRL形式の場合のエレメント名調査
- CSV形式での取得可否

### 2. データモデル設計
- company_propertiesテーブル（EAVパターン）の活用 vs 専用テーブルの検討
- 保持すべきフィールド:
  - 報告者名（holder_name）
  - 保有株式数（shares_held）
  - 保有割合（holding_ratio）
  - 報告理由（変動事由）
  - 提出日（submitted_at）
  - 前回からの変動（delta）
- 報告者のマスタ管理の要否

### 3. インポートジョブ設計
- 既存のImportEdinetDocumentsJobとの統合 or 独立ジョブ
- レート制限の考慮
- 過去データのバックフィル方針

### 4. スクリーニングへの統合
- 「直近N日以内に大量保有報告が出された企業」
- 「特定の投資家が保有比率を増やした企業」
- 「保有比率が閾値を超えた企業」
