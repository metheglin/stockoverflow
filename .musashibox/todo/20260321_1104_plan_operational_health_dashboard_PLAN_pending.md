# PLAN: システム運用状態ダッシュボードの設計

## 概要

データパイプラインの運用状態（データ鮮度、インポート結果、データ品質、エラー状況）を一目で把握できるダッシュボード機能の設計をおこなう。

## 背景・動機

本プロジェクトは個人用のデータベースシステムであり、データの信頼性は分析結果の信頼性に直結する。しかし現在、以下の情報を一元的に確認する手段がない:

- **データ鮮度**: 各データソース（JQUANTS企業マスタ、日次株価、財務データ、EDINET書類）がいつ最後に同期されたか
- **データ量**: 各テーブルのレコード数、直近の増減
- **データ品質**: フィールド充足率（20260321_1103）、メトリクス計算済み率、データ整合性チェック結果
- **パイプライン状態**: 最後のパイプライン実行結果（20260321_1101）、エラー有無
- **カバレッジ**: 上場企業のうちfinancial_values/financial_metricsが存在する割合

ApplicationProperty テーブルに各種同期情報は保存されているが、散在しており、全体像を把握するには複数のレコードを個別に参照する必要がある。

## 設計すべき項目

1. **ダッシュボードの出力形式**
   - Rakeタスク（`rake status:dashboard`）によるコンソール出力を基本とする
   - 将来的にWeb APIに拡張可能な設計
   - 出力はセクション分けされた読みやすいテキスト形式

2. **表示セクション**
   - データ鮮度セクション: 各ソースの最終同期日時・経過時間
   - データ量セクション: テーブル別レコード数・前回比
   - データ品質セクション: フィールド充足率サマリ・計算済み率
   - パイプライン状態セクション: 最終実行結果・エラーサマリ
   - アラートセクション: 対応が必要な問題の一覧

3. **データ収集方法**
   - ApplicationProperty から同期情報を取得
   - 各テーブルの COUNT クエリ
   - FinancialValue::CompletenessAudit の結果活用
   - DataIntegrityCheckJob の結果活用

4. **アラート判定基準**
   - 同期が一定期間（例: 3日）以上滞っている場合
   - データ整合性チェックでissueが検出されている場合
   - 新規上場企業に対してfinancial_valuesが存在しない場合
