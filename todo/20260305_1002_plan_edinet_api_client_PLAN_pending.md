# EDINET APIクライアント設計

## 概要

EDINET API を利用して有価証券報告書等の開示情報を取得するためのAPIクライアントを設計する。
`app/lib/` に配置し、コーディング規約の「汎用性と利便性」の方針に従って設計する。

## 調査・設計の対象

### EDINET APIの仕様調査

- EDINET API v2 の利用可能なエンドポイント
- 認証方式（APIキー: `Rails.application.credentials.edinet.api_key`）
- 書類一覧API、書類取得API の仕様
- XBRLデータの取得・パース方法
- レート制限・利用制約

### クライアント設計

- `app/lib/edinet_api.rb` のクラス設計
  - コーディング規約に従い、`api_key` を引数で指定可能にしつつ、便利メソッド（クラスメソッド）も提供
  - Faraday を使ったHTTPクライアント実装方針
  - エラーハンドリング方針（エラーハンドリング規約に従い、無闇に例外を捕捉しない）
- 取得対象データの選定
  - 有価証券報告書（年次）
  - 四半期報告書
  - 決算短信（対応可能であれば）

### データ取得フロー

- 日次の書類一覧取得 → 対象書類の絞り込み → XBRL取得 → パースの流れ
- XBRLパーサーの実装方針（gem利用 or 自前実装）
