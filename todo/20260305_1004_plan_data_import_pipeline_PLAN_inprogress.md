# データ取り込みパイプライン設計

## 概要

EDINET / JQUANTS から取得したデータをデータベースに取り込み、分析指標を算出するパイプラインを設計する。
`app/jobs/` に配置し、バッチ処理として設計する。

## 設計の対象

### データ取り込みジョブ

1. **企業マスター同期ジョブ**
   - JQUANTS の上場銘柄一覧から企業マスターを同期
   - EDINET の企業コードリストとの紐づけ

2. **決算データ取り込みジョブ**
   - EDINET の書類一覧APIから新着の有価証券報告書・四半期報告書を検出
   - XBRL データの取得・パース・DB保存

3. **株価データ取り込みジョブ**（必要に応じて）
   - JQUANTS から株価四本値の取得・保存
   - バリュエーション指標算出に必要

### 分析指標算出

4. **指標算出ジョブ**
   - 財務数値から各種指標を算出し `financial_metrics` に保存
   - 成長性指標: 売上高YoY、営業利益YoY、経常利益YoY、純利益YoY 等
   - 収益性指標: ROE、ROA、営業利益率、経常利益率 等
   - バリュエーション指標: PER、PBR、PSR 等（株価データが必要）
   - CF指標: 営業CF + 投資CF 符号、フリーCF 等

### 実行スケジュール・運用

- 各ジョブの実行頻度（日次 / 週次）
- 差分更新と全件更新の使い分け
- `application_properties` による最終実行時刻の管理
- エラー発生時の再実行戦略（エラーハンドリング規約に従い、1件の失敗で全体を止めない）
