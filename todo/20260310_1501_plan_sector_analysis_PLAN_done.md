# PLAN: セクター・業種別分析基盤設計

## 背景

`companies` テーブルには `sector_17_code`, `sector_17_name`, `sector_33_code`, `sector_33_name` が格納されており、JQUANTS上場銘柄一覧から業種分類データは取得済みである。

しかし、業種別の集計・比較をおこなう仕組みが一切存在しない。個別企業の ROE が 15% と言われても、その業種の平均 ROE が 5% なのか 20% なのかが分からなければ、企業の優位性を評価できない。

プロジェクトの目的である「注目すべき企業を一覧できるシステム」を高度に実現するためには、業種別の統計量（平均・中央値・四分位）を算出し、個別企業のポジションを把握できる仕組みが必要。

## 設計対象

### 1. 業種別集計指標の算出・保存

- 33業種別 / 17業種別に以下の統計量を算出する仕組み
  - 平均 ROE, ROA, 営業利益率
  - 平均 revenue_yoy, net_income_yoy
  - 中央値 PER, PBR
  - 企業数

### 2. 保存先の設計

- 新テーブル `sector_metrics` を作成するか
- `ApplicationProperty` の kind 拡張で対応するか
- EAVパターンの適用を検討するか
- 算出タイミング（CalculateFinancialMetricsJob の後続として日次？週次？）

### 3. 個別企業の業種内ポジション

- 業種内の偏差値やパーセンタイルを算出するか
- FinancialMetric の data_json に格納するか、別テーブルにするか

### 4. スクリーニングへの活用

- 「業種平均より ROE が高い企業」のような条件でのフィルタリング方法
- 分析クエリレイヤー（plan_analysis_query_layer）との統合方法

## 設計上の考慮事項

- 33業種分類には「銀行業」「保険業」「証券業」など、一般事業会社と財務構造が大きく異なる業種が含まれる。ROE/ROA の比較が意味を成すかは業種によって異なる
- 集計対象は `companies.listed = true` の企業に限定すべき
- 期間：直近の annual 決算のみ対象とするか、四半期も含めるか
