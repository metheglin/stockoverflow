# PLAN: 分析クエリレイヤー設計

## 背景

データ取り込みパイプラインと指標算出ジョブの実装が完了し、`companies`, `financial_values`, `financial_metrics`, `daily_quotes` テーブルにデータが蓄積される基盤が整った。

しかし、蓄積されたデータを実際に分析・活用するためのクエリインターフェースが一切存在しない。プロジェクトの目的である「特定の条件で絞り込みをおこない、注目すべき企業や情報を一覧できるシステム」を実現するために、分析クエリレイヤーの設計が必要。

## 設計対象

CLAUDE.mdに記載されている3つのユースケースを軸に、クエリインターフェースを設計する。

### ユースケース1: 連続増収増益企業の一覧

> 6期連続増収増益の企業を一覧し、増収率が高い順に並べる

- `financial_metrics.consecutive_revenue_growth` および `consecutive_profit_growth` にはDBインデックスが付与済み
- 必要な scope/query: 連続増収増益期数でフィルタ + 最新の revenue_yoy でソート
- 複数期間の情報を結合する必要性

### ユースケース2: キャッシュフロー転換企業の検出

> 営業キャッシュフローがプラスかつ投資キャッシュフローがマイナスの企業のうち、営業キャッシュフローと投資キャッシュフローの差がプラスに転換した企業を一覧する

- 最新期と前期の `financial_metrics` を比較する必要がある
- `operating_cf_positive`, `investing_cf_negative`, `free_cf_positive` は boolean カラムとして存在
- 前期の `free_cf_positive = false` かつ 当期の `free_cf_positive = true` を検出するクエリ

### ユースケース3: 企業の業績転換点分析

> ある企業の業績が飛躍し始める直前にどのような決算・財務上の変化があったかを調べる

- 特定企業の時系列データ取得
- `financial_values` + `financial_metrics` の期間推移を一覧する機能
- YoY変化率のトレンド転換（マイナス→プラス、小→大）の可視化

## 設計すべき項目

1. **モデル scope の設計**
   - `FinancialMetric` に追加すべき scope 一覧
   - `Company` に追加すべき scope 一覧
   - DB Indexの過不足確認

2. **QueryObject クラスの設計**
   - 複数テーブル結合・複数期間比較が必要な複雑クエリは QueryObject として設計
   - クラス名、配置先（`app/models/` 配下）、インターフェースを定義

3. **時系列データ取得インターフェースの設計**
   - 特定企業の財務データ推移を取得するメソッド設計
   - 返却データの構造（Hash? Array of Hash? 専用Value Object?）

4. **汎用フィルタ・ソート機能**
   - 将来追加されるユースケースに対応しやすい設計
   - パラメータによるフィルタリング・ソートの汎用的な仕組み
